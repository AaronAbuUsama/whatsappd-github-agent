# Ship application intake before whatsappd adoption

The current production rollout implements the Conversation Archive, Managed Chat Inbox, and lossless Coalescer inside this application so the stable base does not wait for a cross-repository package change. In parallel, [whatsappd#3](https://github.com/AaronAbuUsama/whatsappd/issues/3) adds the same generic mechanics as an opt-in, framework-neutral `whatsappd/ambient` surface; after that surface is published, [this repository's issue #44](https://github.com/AaronAbuUsama/whatsappd-github-agent/issues/44) adopts it and deletes the application-owned implementation. At every stage there is exactly one production intake path: application-owned now, package-owned later. Managed Chat configuration, Ambience and Flue admission, Skills, tools, and conversational judgment remain application-owned throughout.
