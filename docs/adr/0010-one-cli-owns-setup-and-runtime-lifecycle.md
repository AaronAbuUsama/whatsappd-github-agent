# One CLI owns setup and runtime lifecycle

The published `ambient-agent` package exposes an `ambient-agent` executable using Commander 15 through `@commander-js/extra-typings` for lifecycle commands and Clack prompts for guided `init` and `config` flows. Commander itself requires Node `22.12.0`, while the current Flue runtime and CLI require Node `22.19.0`, so the package declares the effective product floor of `>=22.19.0`. A bare `npx ambient-agent` first run enters setup, while `start` launches the non-interactive foreground runtime and `status` and `doctor` inspect it; the CLI alone resolves the managed data directory and validated configuration, so runtime modules receive dependencies explicitly and do not read environment variables. OS-level background supervision remains the deployment platform's responsibility in this rollout.
