# Managed files own local configuration and credentials

The `ambient-agent` executable owns one OS-appropriate managed data directory containing validated non-secret configuration, an application SQLite database, a Flue SQLite database, WhatsApp credentials, and provider credential files protected by directory mode `0700` and file mode `0600`; normal operation does not require user-supplied environment variables. Configuration refers to credentials by stable names rather than embedding secret values, Pi OAuth uses an application-selected auth-storage path inside the directory, the composition root loads both once, and runtime modules receive typed dependencies without exposing raw credentials to the agent sandbox. A future OS-keychain adapter may replace file-backed credential storage without changing the configuration interface.
