# Managed files own local configuration and credentials

For the SaaS tenant runtime, ADR 0018 supersedes this decision only for WhatsApp and ChatGPT credential storage when the complete tenant database environment contract is configured.

The `ambient-agent` executable owns one OS-appropriate managed data directory containing validated non-secret configuration, an application SQLite database, a Flue SQLite database, WhatsApp credentials, and provider credential files protected by directory mode `0700` and file mode `0600`; normal operation does not require user-supplied environment variables. Configuration refers to credentials by stable names rather than embedding secret values. Ambient Agent stores the complete ChatGPT OAuth record at `credentials/chatgpt-oauth.json`, while Pi remains a private model adapter rather than the operator-facing credential owner. The CLI composition root loads managed dependencies once and installs the typed application-factory input before importing Flue's generated server; the generated app receives that dependency explicitly without exposing raw credentials to the agent sandbox. A future OS-keychain adapter may replace file-backed credential storage without changing the configuration interface.
