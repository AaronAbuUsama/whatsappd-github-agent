## 1. Why this phase exists

§13's own closing paragraph defines the remaining work, and it is one coherent theme:

> "The remaining correction is to finish **concentrating routing, work, and proactive time in the
> Brain**, then remove the superseded Speaker and routing ownership paths. The remaining distance is
> **concentration of authority** and honest delivery boundaries, not a parallel replacement platform."

This PRD is derived from §13's "Distance" column — not from the current code's limitations. That
distinction matters: an earlier draft of this plan treated implementation shortcuts
(`file_issue`'s surface-resolved repo, the `surfaceRepositories` config field) as if they were the
design. They are **drift to remove**, not constraints to build around.

**Routing is the Brain's, always:**
- §8:529 — "The Brain chooses **surface and voice** as part of every decision" — including saying it
  in a group *and* continuing a DM, with different messages.
- §11:592 — "No new routing concept is needed — routing is '**the Brain decides**,' and it already does."
- §11:601-604 — "**Multiple projects per surface, or multiple surfaces per project.** … the mapping of
  'which surface cares about which project' is **data in the Graph, not hard-wired configuration —
  the Brain resolves it per decision**."

**Config is authorization, never routing** (§8:533): "Configured groups remain operator-authorized.
Discovery alone never grants participation." `managedChats` / `allowedRepositories` are the
fail-closed permission boundary; *which* surface hears *what* about *which* repo is a Brain decision
over Graph data.

## 5. Proof contract (ratified)

Each spec declares which layers are mandatory.

| Layer | Meaning |
|---|---|
| **L1** | WhatsApp receipt — nonce-tagged message from the independent driver account, provider message ID |
| **L2** | Archive/DB rows in `application.sqlite` — Intent / Batch / Effect / Delivery, `event_id` correlated |
| **L3** | GitHub artifact — real URL authored by `ambient-*[bot]`, never a human login |
| **L4** | Braintrust trace — Brain spans correlated by `evidenceIds == event_id` (live since 2026-07-23) |
| **L5** | Restart-survival — byte-identical receipt across `systemctl restart` |
| **L6** | Screenshot/behavioral — where rows can't show it (typing, preamble timing, transcript) |

Every spec also states a **negative assertion** — the failure the proof must actively exclude.

## 6. Work items

### S1 — Route GitHub events through the Brain-owned up-inbox  · **#249** (existing)
**Closes:** §13 Control loop + Brain (routing half).
**Problem:** `ingress.ts:124` "A supported GitHub event broadcasts to all of them"; `:515` broadcasts
to every managed thread; §13 — "uncorrelated events are still **dropped**". Violates *No silent drop*
and leaks across companies.
**Build:**

- Remove the routing shortcuts: `file_issue`'s "resolved from that Surface — never chosen here"
  (`packages/agents/src/brain/tools.ts:64`) and the `surfaceRepositories` config field. Repo is
  resolved by the Brain per decision from Graph relations (§11:601-604) — needs S2.

**Proof:** L1 · L2 · L3 · L4 · L5.
**Negative:** an event correlated to org A **never** appears in org B's chat; an uncorrelated event
still lands in the inbox (zero drops).
**Deps:** S2. **Gates:** S6.

**Closes:** §13 Digest distance. **Ships first.**

  chatter** — never an acked request. *(Aaron, 2026-07-23.)*

**Proof:** L1 · L2 · L4 · L5 · L6.
**Negative:** **no acked request is ever left unclosed**; no silent gap on a slow action.
**Deps:** none. Related: #316.

### S4 — The proactive clock · **NEW ISSUE**
**Closes:** §13 Two clocks; #299 "Proactive liveness".
**Problem:** "Only the reactive clock exists. Overdue commitments are *flagged* in the digest but
**nothing acts on them**" (§13). No proactive implementation exists in code.
**Build:** **Scheduled Wake** (durable exact reconsideration) + coalesced **Proactive Sweep** (§6,
glossary): one sweep when none is outstanding, admitting every due wake; reuse the durable-ledger +
boot-sweep pattern; support self-scheduled wakes. Initiative — chasing an overdue commitment — lives
here.
**Proof:** L1 (an overdue commitment produces a real chase message) · L2 (wake ledger rows) · L4 · L5.
**Negative:** **no duplicate sweep** while one is outstanding; a due wake survives restart and fires
exactly once.
**Deps:** none (S3 makes its output legible).

### S5 — Known-Person Surface resolution · **NEW ISSUE**
**Closes:** §13 Surfaces distance; #299 "one chat routing path for configured group **and direct** chats".
**Problem:** §13 — "The Flue Speaker id is still the provider chat JID and **known-Person target
resolution is absent**." A real DM (`204663831932940@lid`) was archived but never acted on;
`packages/engine/src/coalescer/chat-gate.ts:19` is an exact-JID allowlist with "no broad DM escape hatch."
**Build:** a Brain prompt targets **either** a stable Surface **or a known Person**, whom trusted code
resolves to the same ordinary Surface registry during prompt admission (§8:532). Remove the remaining
provider-chat-JID routing shortcut. "DM someone" and "reply in the group" share **one** prompt operation.
**Preserve fail-closed:** configured groups stay operator-authorized; discovery alone never grants
participation (§8:534, §10).
**Proof:** L1 (one decision speaks in a group *and* DMs a person, with **different messages**) · L2 ·
L4 · L5 · L6.
**Negative:** an **unknown** person's DM still lands no participation — observation never grants it.
**Deps:** S1 (conceptually).

### S8 — Live-reload the authorization config · **#179** (existing)
**Closes:** operational friction. Scoped to **authorization only**.
**Problem:** `readManagedConfig` is boot-only; `makeManagedChatGate(managedChats)` is built once

stream is untouched).
**Negative:** a WhatsApp/model/port change **still requires a restart** and is never silently
hot-swapped; a reload never re-establishes the WhatsApp connection.
