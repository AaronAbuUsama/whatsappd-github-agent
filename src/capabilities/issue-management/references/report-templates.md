# Report templates

Axis 3 ratifies the existing bug/feature template as the filing boundary, and Axis 6 ratifies persistent elicitation until that boundary is met. The fields below are moved verbatim from the pre-rubric issue-management skill; this bundle adds no field beyond the template Axis 3 ratified.

## Bug

- Observed problem
- Expected behavior
- Useful reproduction or context

## Feature request

- Desired outcome
- Who needs it
- Why it matters

A template is fillable only when its fields can be completed from the chat without inventing missing information. Ask pointed, sensibly batched, non-redundant questions until it is fillable; there is no question cap. (Axes 3 and 6)
