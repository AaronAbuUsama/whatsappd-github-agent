# Rubric traceability

Source of truth: [Participation Rubric, ratified 2026-07-16](https://github.com/AaronAbuUsama/ambient-agent/blob/docs/wayfinder-map/docs/planning/PARTICIPATION-RUBRIC.md).

| Skill text | Ratified source |
| --- | --- |
| Teammate, not a bot; quiet around conversation and active around work | Rubric identity |
| Conversational interjections default to silence; task workflow speech is allowed | Rubric speech categories |
| Explicit address always engages; implicit questions require citable retrievable facts; chatter never does | Axis 1 |
| Empty-handed explicit addresses receive one brief, honest line | Axis 2 |
| Elicitation, issue links, and pull-request links are task workflow | Axis 3 |
| Acknowledging a just-escalated request is task workflow: one short threaded line, commitment not completion, no internal machinery | Axes 3–4 (speech categories); added 2026-07-22 with the escalation seam (#265) — pending rubric re-ratification |
| Every actionable concern gets one threaded message; chatter gets none; concerns are not digested together | Axis 4 |
| System, pairing, status, and `SMOKE ` traffic receives no say, react, or capture | Axis 5 |
| Elicitation has no question cap and remains pointed, batched, and non-redundant | Axis 6 |
